## 🔴 TIER S — 绝对核心（综合评分 ≥ 13/15）

> 不装这10个，不开始挖洞

### 1. PayloadsAllTheThings
- **链接**: https://github.com/swisskyrepo/PayloadsAllTheThings
- **Stars**: 65k+
- **唯一价值**: 原理 + Payload + 绕过技巧三合一，无任何替代品
- **用法**: 打不动时查对应漏洞类型，找绕过思路

### 2. nuclei + nuclei-templates
- **引擎**: https://github.com/projectdiscovery/nuclei
- **模板库**: https://github.com/projectdiscovery/nuclei-templates
- **Stars**: 22k+ / 10k+
- **唯一价值**: 9000+ 模板，一条命令覆盖所有已知漏洞检测
- **用法**: `nuclei -u https://target.com -t templates/`

### 3. interactsh
- **链接**: https://github.com/projectdiscovery/interactsh
- **唯一价值**: OOB 交互平台，Blind SSRF / Blind XSS / SSRF → RCE 唯一验证方式
- **用法**: 生成 OOB payload，监听回调

### 4. subfinder
- **链接**: https://github.com/projectdiscovery/subfinder
- **唯一价值**: 被动子域名枚举速度/覆盖率行业第一，纯被动零发包
- **用法**: `subfinder -d target.com -all -recursive`

### 5. httpx
- **链接**: https://github.com/projectdiscovery/httpx
- **唯一价值**: 所有工具输出的 HTTP 验证层，探活 + 指纹 + 状态码
- **用法**: `cat subdomains.txt | httpx -title -tech-detect -status-code`

### 6. trufflehog
- **链接**: https://github.com/trufflesecurity/trufflehog
- **唯一价值**: 多源 Secret 扫描（Git / S3 / CI/CD / 云存储），经常直接出 Critical
- **用法**: `trufflehog github --org=target --only-verified`

### 7. SecLists
- **链接**: https://github.com/danielmiessler/SecLists
- **Stars**: 60k+
- **唯一价值**: 唯一纯字典库，Fuzz / 目录爆破 / 参数枚举离不开它
- **用法**: ffuf / nuclei / sqlmap 的弹药库

### 8. gau (getallurls)
- **链接**: https://github.com/lc/gau
- **唯一价值**: 拉取 Wayback / OTX / CommonCrawl 历史 URL，katana 爬不到过去的数据
- **用法**: `gau target.com | grep "\.php\|\.asp\|api"`

### 9. BBOT
- **链接**: https://github.com/blacklanternsecurity/bbot
- **唯一价值**: 递归模块化侦察，发现其他工具遗漏的资产，2024-2025 最强框架
- **用法**: `bbot -t target.com -p subdomain-enum`

### 10. hackerone-reports
- **链接**: https://github.com/reddelexc/hackerone-reports
- **唯一价值**: H1 官方按赏金排序的公开报告，学高赏金思路唯一权威来源
- **用法**: 按漏洞类型筛选，读 Critical/High 报告

## 🟠 TIER A — 高价值工具（综合评分 10-12/15）

### 11. ffuf
- **链接**: https://github.com/ffuf/ffuf
- **价值**: 速度最快 Fuzz 引擎，目录/参数/VHost 模糊测试，灵活性无替代
- **用法**: `ffuf -u https://target.com/FUZZ -w SecLists/Discovery/Web-Content/common.txt`

### 12. sqlmap
- **链接**: https://github.com/sqlmapproject/sqlmap
- **价值**: SQLi 完整利用链（检测 → 数据提取 → GetShell），nuclei 只检测不利用
- **用法**: `sqlmap -u "https://target.com/page?id=1" --dbs --batch`

### 13. dalfox
- **链接**: https://github.com/hahwul/dalfox
- **价值**: XSS 深度测试，误报率远低于 nuclei，支持 DOM XSS
- **用法**: `dalfox url "https://target.com/search?q=test"`

### 14. katana
- **链接**: https://github.com/projectdiscovery/katana
- **价值**: 现代 JS 渲染爬取，发现 SPA 应用隐藏端点
- **用法**: `katana -u https://target.com -js-crawl -d 3`

### 15. reconftw
- **链接**: https://github.com/six2dez/reconftw
- **价值**: 集成 50+ 工具的全自动侦察，10 分钟启动完整流程
- **用法**: `./reconftw.sh -d target.com -r`

### 16. gf
- **链接**: https://github.com/tomnomnom/gf
- **价值**: 参数语义过滤，从 10 万条 URL 里识别 sqli/ssrf/redirect 危险参数
- **用法**: `cat urls.txt | gf ssrf`

### 17. uncover
- **链接**: https://github.com/projectdiscovery/uncover
- **价值**: 聚合 Shodan + Fofa + Censys 单次查多源测绘数据
- **用法**: `uncover -q 'ssl:"target.com"' -e shodan,fofa,censys`

### 18. dnsx
- **链接**: https://github.com/projectdiscovery/dnsx
- **价值**: subfinder 输出的 DNS 验证层，批量解析/验证/枚举
- **用法**: `cat subdomains.txt | dnsx -resp`

### 19. HowToHunt
- **链接**: https://github.com/KathanP19/HowToHunt
- **价值**: 每类漏洞具体挖法 SOP，新方向必查
- **用法**: 遇到不熟悉的漏洞类型时查操作步骤

### 20. Google-Dorking-2025
- **链接**: https://github.com/abu76/Google-Dorking-for-Bug-Bounty-
- **价值**: 2025 最新 Dork 列表，零发包被动信息收集
- **用法**: 复制 Dork 到 Google，配合 `site:target.com`

### 21. daily-bugbounty-writeups
- **链接**: https://github.com/securitycipher/daily-bugbounty-writeups
- **价值**: 每日实时更新，追踪最新漏洞手法，防止知识过时
- **用法**: 每天花 10 分钟读一篇最新 writeup

### 22. smuggler
- **链接**: https://github.com/defparam/smuggler
- **价值**: HTTP 请求走私专用工具，无其他工具覆盖此方向
- **用法**: `python3 smuggler.py -u https://target.com`

## 🟡 TIER B — 垂直专项（特定场景不可替代）

### 23. naabu
- **链接**: https://github.com/projectdiscovery/naabu
- **场景**: 目标有非标准端口 / 网段级扫描
- **用法**: `naabu -host target.com -p - -silent | httpx`

### 24. asnmap
- **链接**: https://github.com/projectdiscovery/asnmap
- **场景**: 挖大厂，需要找所有 IP 段
- **用法**: `asnmap -org "Target Corp" -json`

### 25. cvemap
- **链接**: https://github.com/projectdiscovery/cvemap
- **场景**: 发现特定组件版本，快速查对应 CVE
- **用法**: `cvemap -product nginx -version 1.18`

### 26. nomore403
- **链接**: https://github.com/devploit/nomore403
- **场景**: 遇到 403/401 时专项绕过
- **用法**: `nomore403 -u https://target.com/admin`

### 27. rengine
- **链接**: https://github.com/yogeshojha/rengine
- **场景**: 团队协作 / 持续监控 SRC 项目
- **用法**: Web UI 配置，自动化定期扫描

### 28. Bug-Bounty-Hunting-Methodology-2025
- **链接**: https://github.com/amrelsagaei/Bug-Bounty-Hunting-Methodology-2025
- **场景**: 不确定工具链时查阅，2025 最新流程参考
- **用法**: 新目标时对照方法论检查遗漏步骤

### 29. OWASP/API-Security
- **链接**: https://github.com/OWASP/API-Security
- **场景**: 目标是纯 API 服务时
- **用法**: 对照 API Top 10 逐项测试

### 30. slither
- **链接**: https://github.com/crytic/slither
- **场景**: Web3 / 智能合约审计项目
- **用法**: `slither contract.sol`

### 31. prowler
- **链接**: https://github.com/prowler-cloud/prowler
- **场景**: 目标有云基础设施（AWS/GCP/Azure）
- **用法**: `prowler aws --region us-east-1`

### 32. Awesome-Hacking (索引)
- **链接**: https://github.com/Hack-with-Github/Awesome-Hacking
- **场景**: 需要扩展某个方向工具时的导航索引
- **用法**: 按安全方向查找更多专项工具
