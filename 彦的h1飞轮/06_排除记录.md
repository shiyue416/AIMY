# 排除记录 — Negative Findings

> 记录"什么不成立"和记录"什么成立"一样值钱。
> 每条排除 = 省掉未来重复探索的时间 + Token。

---

## 格式

```
日期：YYYY-MM-DD
目标：target.com
假设：简短描述怀疑的漏洞类型和位置
测试：实际怎么测的（参数/请求/工具）
结果：返回了什么
排除原因：为什么这个假设不成立（后端校验/过滤/WAF/配置）
关联：哪个阶段/信号触发的这条测试
```

---

## 用途

| 场景 | 怎么用 |
|------|--------|
| **Phase 4 卡壳** | 翻排除记录 → 避免重复试同一个方向 |
| **重测同一目标** | 查看上次排除了什么 → 直接跳过已排除路径 |
| **省 Token** | AI 不用重新探索已确认不存在的方向 |
| **提升置信度** | 排除记录越厚，剩下的发现越值得深挖 |
